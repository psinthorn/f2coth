## Bugs found

### P0 — must fix before production
1. **Seed admin password hash is a placeholder.**
   `database/migrations/007_seed_data.sql` ships a bcrypt hash that was hand-written for the literal string `F2@admin2026`, but it was *not* generated by an actual bcrypt run. The user will not be able to log in with that password until the hash is regenerated.
   **Fix:** run `htpasswd -bnBC 12 "" 'F2@admin2026' | tr -d ':\n'` (or any bcrypt cost-12 generator) and replace the `password_hash` value. Or, simpler: delete the seeded admin row and add a one-shot CLI command to create the first admin user.
   **File:** `database/migrations/007_seed_data.sql:11`

### P1 — should fix before launch
2. **No `go.sum` files committed.** `Dockerfile` runs `go mod download`, which will succeed without `go.sum` only if Go's `GOFLAGS=-mod=mod` (the default), but this means each fresh build re-resolves dependencies and can drift. Run `make tidy` once a Go toolchain is available, then commit the resulting `go.sum` files.
   **File:** all 5 service `go.mod` files
3. **No `package-lock.json` for the Next.js app.** Same drift risk. Run `npm install` once, commit `package-lock.json`.
   **File:** `services/web-app/package.json`
4. **React 19 RC pin may be stale by the time the user runs `npm install`.** The version `19.0.0-rc-66855b96-20241106` is an exact RC tag; it may need updating to a stable release.
   **Fix:** `cd services/web-app && npm install react@latest react-dom@latest next@latest` and re-pin.

### P2 — nice to have
5. **`/admin/login` does not explicitly set `robots: { index: false }`** because client components can't export `metadata`. Recommend either making it a server component shell with `metadata`, or adding `<meta name="robots" content="noindex" />` via `head.tsx`.
6. **CORS allow-list defaults to `*`.** That's fine for local dev, but the `.env.example` should make the production lockdown more obvious. Already documented as `https://f2.co.th` in the example, but a comment would help.
7. **No rate limiting on public endpoints.** Contact form and chat endpoints accept unlimited requests per IP. Add a Traefik middleware (rate-limit) before launch.
8. **Login audit log writes use `r.RemoteAddr` directly into `INET`.** With Traefik in front this will be `traefik_container_ip:port`, not the client IP. The middleware chain already runs `chimw.RealIP`, so `r.RemoteAddr` will reflect the X-Forwarded-For after that — verify in dynamic testing.

## Suggested fixes

- **#1** is the single blocker. Regenerate the bcrypt hash on a real machine before declaring "the admin can log in".
- **#2/#3** are tooling chores; both resolve themselves on the first real build.
- **#5–#8** are pre-launch cleanups, not pre-merge blockers.
