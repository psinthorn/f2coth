AssetHub is F2’s multi-tenant IT asset register, built into the main platform. Collector scripts and a network probe push inventory over outbound HTTPS; staff triage and enrich it, then hand it back to the client as a branded document. Here is the whole flow, captured from the running app.

Flow Issue token→ Collect & probe→ Triage→ Enrich→ Hand over
1

The asset register

One register per client organization. Headline counts up top; section chips split the same dataset into Network & Wi-Fi, Computers, CCTV and Printers; search and CSV export sit inline.

2

Issue a token, hand over the command

Mint a per-site enrollment token — shown once — and the install panel fills it straight into ready-to-run one-liners. The collectors and probe download from the server itself, so a technician needs nothing but the command.

3

Triage what the probe found

The network probe’s findings arrive here with a guessed type. Classify and Promote a device into the register, or Ignore the noise. MAC matches to known devices are linked automatically.

4

Inspect and enrich a device

Everything one machine reported — Hardware, Network, Software, History — alongside the fields an engineer owns: asset tag, assigned user, site and status.

5

Hand it back as a document

Generate the client handover as XLSX, PDF or DOCX. A background worker renders it (Thai-capable), and the file downloads the moment it’s ready.

6

Self-enroll from a phone

For phones, tablets and CCTV that no script can reach: a login-free form — the token rides in the URL — registers a device in under a minute.
